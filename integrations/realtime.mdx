CityOS uses **Ably** for all client-facing realtime: IoT sensor readings, fleet driver positions, booking status updates, and event-gate scans. Channels are tenant-scoped and short-lived auth tokens are issued by the BFF.

## Auth

Call `/api/bff/tenant/ably-token` to mint a tenant-scoped Ably token. The BFF embeds the user's roles so Ably enforces per-channel ACLs.

```typescript
import Ably from "ably";

const ably = new Ably.Realtime({
  authUrl: `${baseUrl}/api/bff/tenant/ably-token`,
  authHeaders: {
    Authorization: `Bearer ${await tokenStore.get()}`,
    "x-tenant-slug": "riyadh-downtown",
  },
});
```

The Ably client auto-refreshes the token before expiry.

## Channel naming

All channels follow:

```text
cityos:{tenantSlug}:{domain}:{topic}
```

## Reconnection & presence

The Ably client handles reconnection transparently. For presence channels (fleet drivers, support agents), call `presence.enter()` on connect and `presence.leave()` on shutdown:

```typescript
const presence = ably.channels.get("cityos:riyadh-downtown:fleet:presence");
await presence.presence.enter({ driverId, shift: "morning" });

// later
await presence.presence.leave();
```

## Rate limits

The BFF issues tokens with these defaults (configurable per tenant):

| Limit | Default |
| --- | --- |
| Concurrent connections per user | 10 |
| Messages per second per channel | 100 |
| Token TTL | 60 minutes |

## Capability flag

Gate every Ably subscription on `ablyEnabled`:

```typescript
const { data } = await tenant.getCapabilities();
if (!data.ablyEnabled) {
  console.warn("Realtime not available; falling back to polling");
  return;
}
```
