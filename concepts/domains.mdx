Dakkah CityOS organizes all business logic into domain packages. Each package is an independent TypeScript library that follows Domain-Driven Design (DDD) principles. A domain package owns its collections, SDUI blocks, React hooks, and BFF API routes. This means that the code for restaurant menus, citizen complaints, parking zones, healthcare appointments, and fleet telemetry live in entirely separate packages — they do not share state, and they do not call each other directly.

There are over 77 domain packages in CityOS. Every package that exposes public API routes does so under `/api/bff/<domain>/`, following the BFF (Backend for Frontend) pattern.

## Domain package structure

Every domain package is a self-contained TypeScript library that exports React hooks, SDUI block definitions, and BFF API routes for its vertical. Each package exports a `domainPackage` object that the platform's domain registry uses to discover available features at boot — you do not configure domain registration manually.

## BFF route conventions

All domain API routes follow a consistent pattern:

```
/api/bff/<domain>/<resource>
/api/bff/<domain>/<resource>/[id]
/api/bff/<domain>/<resource>/[id]/<action>
```

Every BFF route automatically enforces authentication, tenant context resolution, and role-based access before your handler runs. You never need to add manual permission checks inside your client code.

```http
# List products in the commerce domain
GET /api/bff/commerce/products
x-tenant-slug: riyadh-municipality
Authorization: Bearer <your-jwt>

# Get a single order
GET /api/bff/commerce/orders/ord_01HXYZ
x-tenant-slug: riyadh-municipality
Authorization: Bearer <your-jwt>

# Submit a citizen complaint
POST /api/bff/citizen/complaints
x-tenant-slug: riyadh-municipality
Authorization: Bearer <your-jwt>
Content-Type: application/json

{ "category": "road-maintenance", "description": "Pothole on King Fahd Road", "nodeId": "zone-3-riyadh" }
```

<Note>
Not all 77 domains have BFF routes fully implemented yet. Domains that are CMS-only (no external data source) may not expose a `/api/bff/` route. Check the domain package's `src/api/` directory to confirm route availability before building an integration.
</Note>

## Using domain hooks in your app

Domain packages export typed React hooks that abstract BFF calls. These hooks handle loading state, error handling, and data normalization for you:

```typescript
import { useProducts } from "@cityos/domain-commerce";
import { useOrders } from "@cityos/domain-commerce";

function ProductList() {
  const { products, loading, error } = useProducts({ tenantSlug: "riyadh-municipality" });

  if (loading) return <Spinner />;
  if (error) return <ErrorBanner message={error.message} />;

  return products.map((product) => <ProductCard key={product.id} product={product} />);
}
```

Each domain hook is self-contained. You do not need to configure a global store or provider — hooks source their data directly from the BFF layer using the tenant context already present in your session.
