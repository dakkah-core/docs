### TenantCurrencyResolver

**Kind:** type | **File:** `block-currency.ts`
> Block-level currency formatter helper. -------------------------------------- Canonical way for a block to render a monetary amount. Resolves from: 1. Explicit `currency` prop on the block 2. Tenant default (when a resolver is installed) 3. Locale default (en → USD, fr → EUR, ar/ar-SA → SAR) 4. Hard-default 'SAR' Kept in @cityos/blocks-core to avoid every block package peer-depending on @cityos/i18n / @cityos/design-runtime directly.

```ts
type TenantCurrencyResolver = () => string | undefined;
```

**Kind:** interface | **File:** `block-data.ts`
> Block data-adoption helper. -------------------------- Canonical pattern for blocks that accept either SSR-prefetched `initialData` OR an async hook result. The helper collapses both sources into a single `{ data, loading, error }` shape so blocks can render with one branch: const { data, loading, error } = useBlockData({ initialData: props.initialData, fetched, }) When `initialData` is provided, the hook is effectively inert — `loading` is false and `error` is null.
